# Operator guide

How to use the WitUS Triage Agent — written for the person running the queue, not
for developers. No coding, no admin setup. If you can sign in, you can use everything
here.

> The in-app version of this guide lives at **`/help`** in the running app. These
> markdown files are the source of truth; keep the two in sync when the app changes.
> (These are engineering/operator notes, so the APA-7 citation rule does not apply —
> see `CLAUDE.md`.)

## Contents

1. [Getting started](00-getting-started.md) — sign in, reach the queue, the 60-second model.
2. [The queue](01-the-queue.md) — the to-do list, statuses, History.
3. [Reading a run](02-reading-a-run.md) — classification, context, proposed action, audit trail.
4. [Approving, rejecting, editing](03-approving-rejecting-editing.md) — your decision controls.
5. [Categories & actions](04-categories-and-actions.md) — what the labels mean.
6. [FAQ & troubleshooting](05-faq-and-troubleshooting.md) — failures, offline, getting help.

## Who this is for

Anyone the operator invites to help triage incoming messages. You review what the agent
proposes and make the call — the agent never sends anything on its own.

Need the engineering side instead? See the repo [`README.md`](../../README.md) and the
[`docs/lessons/`](../lessons/README.md) curriculum.
