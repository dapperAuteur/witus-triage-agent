# Classification accuracy — EVAL.md

This file is regenerated by `npm run eval` (`__tests__/agent/accuracy.test.ts`), which
runs the `classify` node against the 25 hand-labeled fixtures in
`__tests__/fixtures/submissions.json` and measures accuracy against the PRD's ≥ 80% bar
(acceptance criterion #5).

## Status: not yet measured on a clean run

The most recent attempt (2026-05-18, model `gemini-2.5-flash`) was **blocked by the
Gemini free-tier daily request quota**, not by classifier quality:

- Every fixture after the first returned a `429 Too Many Requests — quota exceeded`
  error. The `classify` node is fail-soft, so each blocked call became its `other` /
  confidence-0 fallback. That produced an unrepresentative 8% — a quota artifact, not a
  measurement.
- The one call that completed before the quota ran out (`fixture-01`) classified
  correctly with confidence 1.0, and the `classify` test suite scores 5/5 on its sample
  set whenever quota is available.

## How to get a real number

Run the eval on fresh quota — the Gemini free-tier daily cap resets at midnight Pacific
— or with a funded `ANTHROPIC_API_KEY` (`TRIAGE_LLM_PROVIDER=anthropic`):

```bash
npm run eval
```

The eval paces its calls (`EVAL_DELAY_MS`, default 7s) to stay under the per-minute
limit, so a clean daily quota is the only remaining requirement. The run overwrites this
file with the dated accuracy report — overall, by category, and per fixture.
